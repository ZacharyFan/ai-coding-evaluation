# 评分规则

[英文版](rubric.md)

盲审时使用这个评分规则。Reviewer 不应该知道该 run 是哪个工作流产出的。

## 正确性

```text
1.0 完整解决任务，并处理已声明的边界情况
0.7 解决主路径，但存在轻微边界遗漏
0.4 只部分解决任务
0.0 没有解决任务
```

## 回归安全性

```text
1.0 保持现有行为，相关测试通过，没有 API/UX 破坏
0.7 低风险变更，但有轻微不确定性
0.4 存在明显回归风险，或证据不足
0.0 已知破坏
```

## 可维护性

```text
1.0 简单、局部、符合项目习惯、容易 review
0.7 基本合理，但有可避免的复杂度
0.4 难维护，或 diff 范围过大
0.0 难以阅读或非常脆弱
```

## 测试质量

```text
1.0 测试覆盖核心行为，并能防住未来同类回归
0.7 测试覆盖主路径
0.4 测试较浅，或断言较弱
0.0 没有有意义的测试
```

## 安全性

```text
1.0 未发现安全问题
0.7 有轻微加固空间
0.4 存在明显未解决风险
0.0 引入安全漏洞
```

## 过程合规性

这是稳定性预测信号，不替代正确性。它衡量 agent 的工作方式是否能在更难任务上复现。

```text
1.0 使用了正确的项目指令、文档、工具，并保留自检轨迹
0.7 基本遵循预期流程，但证据不完整
0.4 代码可能正确，但过程证据弱，或像是碰巧正确
0.0 忽略关键项目指令、文档、工具或 review 要求
```

## 效率

根据任务预算计算：

```text
duration_minutes
human_interventions
cost_usd
```

低于预算时接近 `1.0`；超过预算后按比例衰减。
